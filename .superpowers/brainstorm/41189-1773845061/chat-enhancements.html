NEW: Chips generated from response context

How chips are generated

The AI response includes a structured suggested_replies field. The LLM system prompt instructs it to suggest 2-3 natural follow-ups based on the conversation context.

LLM system prompt JSON parsing Fallback to static

Chip types

Explore deeper: "Other side effects?", "Is this normal?"
Take action: "Log symptom", "Talk to doctor"
Navigate: "View my timeline", "Check medicines"

Fallback behavior

If LLM doesn't return chips (or mock mode), fall back to context-based rules: symptom mentioned → offer "Log symptom", medicine mentioned → offer "Side effects?", etc.

Session data model

New ChatSession SwiftData model: id, title (auto-generated from first user message), createdAt, profileId. ChatMessage gets a sessionId field.

SwiftData @Model Migration

Auto-title generation

Session title = first user message, truncated to 40 chars. Can optionally ask LLM for a better title after 3+ messages.

Multi-turn context

When loading a session, pass last 10 messages as context to LLM. This gives the AI memory within a conversation.

Context window management

New session triggers

Auto-create new session: first message of the day, or manually via "New Chat" button. Old sessions are read-only browsable.
