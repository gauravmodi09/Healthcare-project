Complete SupabaseService.swift — real initialization, auth flow (Apple Sign In + email), session management. This unblocks everything: cloud sync, meeting pipeline, vector search, push notifications.

  • • Initialize Supabase client with real project URL + anon key
  • • Build login/signup screens (Apple Sign In + email magic link)
  • • Session persistence + token refresh
  • • Wire AppService to call edge functions instead of local-only
  • • Add .env / Secrets.plist entries for Supabase credentials

P0 — Do First

Complete Meeting Recording Pipeline

Wire the full flow: Record audio → Upload to Supabase Storage → Call transcribe-meeting edge function → Display structured notes → Auto-create tasks from action items. This is the #1 "wow moment" feature.

  • • Upload recorded audio to Supabase Storage after stop
  • • Call transcribe-meeting edge function
  • • Parse response → populate MeetingNotes view
  • • Auto-create tasks from action_items where is_mine=true
  • • Show "Saved to memory" confirmation bar
  • • Delete audio after transcription confirmed
AudioRecorderService Storage + Edge Fn MeetingNotesView
P1 — Core Loop

Cloud Sync + Offline Queue

Sync local GRDB data to Supabase. Capture locally → queue for upload → sync on connectivity. Pull cloud data on launch. Last-write-wins conflict resolution.

  • • Create SyncService with upload queue
  • • On capture: write GRDB + enqueue Supabase insert
  • • Background sync on app launch + every 5 min
  • • Pull cloud changes to local on sync
  • • Handle offline gracefully (queue up to 100 items)
SyncService Supabase Realtime GRDB queue
P1 — Core Loop

Wire Vector Search to UI

Connect MemorySearchView to search-memory edge function. Generate embeddings on capture. Show AI synthesis + source cards. This makes the "searchable life memory" promise real.

  • • On new capture → call generate-embedding edge function
  • • Search bar → call search-memory edge function
  • • Display AI synthesis answer
  • • Show source memory cards with dates
  • • Fallback to local GRDB search when offline
